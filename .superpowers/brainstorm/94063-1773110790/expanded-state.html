A. Inline Expansion

The bottom sheet itself expands to show full venue detail content (photos, 3-axis ratings, reviews, community posts) within the sheet. Scrollable inside. No page navigation needed. The compact card becomes a sticky header.

B. Navigate to Detail Page

Swipe up past a threshold triggers router.push('/venue/[id]') — navigates to the existing full venue detail page. Sheet is only the preview layer. Back button returns to map. Simpler to build, reuses existing page.

Option A — Inline

  • Seamless, no page transition
  • User stays on map context
  • Fluid Apple Maps feel
  • More complex to build (duplicate detail content in sheet)

Option B — Navigate

  • Reuses existing venue/[id] page as-is
  • Less code to write/maintain
  • Clear mental model (preview → detail)
  • Page transition breaks fluid feel slightly
